1. **mapGenerator.ts**: Create Act 1 map generation.

Export types:
```typescript
export type RoomType = 'encounter' | 'elite' | 'event' | 'campfire' | 'treasure' | 'merchant' | 'boss';
export interface MapNode {
  id: string;           // e.g., "1-0", "2-1" (floor-position)
  floor: number;        // 0-based floor number
  type: RoomType;
  connections: string[]; // IDs of nodes this connects to (on next floor)
}
export interface GameMap {
  nodes: MapNode[];
  bossNodeId: string;
  currentNodeId: string | null; // null = not yet on map (at Neow)
}
```

Export `generateMap(rng: () => number): GameMap`:
- Generate a 15-floor Act 1 map (floors 0-14, floor 14 = boss)
- Floor 0: 1st encounter (always encounter type) — 2-4 starting nodes based on player count conceptually, but use 3 fixed start nodes
- Floors 1-13: 3-4 nodes per floor with branching paths
- Floor 14: 1 boss node
- Room type distribution (approximate board game feel):
  - Floor 0: always encounter (first_encounter)
  - Floors 1-5: mostly encounters, some events
  - Floor 6: treasure room (guaranteed)
  - Floor 7-8: elite possible, events, encounters
  - Floor 9: campfire (guaranteed rest before harder content)
  - Floors 10-12: mix of encounters, elites, events, merchants
  - Floor 13: campfire (guaranteed rest before boss)
  - Floor 14: boss
- Each node connects to 1-2 nodes on the next floor
- Ensure all nodes are reachable (no orphans)
- Use RNG for randomization (testable)

2. **gameInit.ts**: Export `initializeGame(room: Room, rng?: () => number): CombatGameState`:

Step-by-step initialization:
a. **Starter decks**: For each player in lobby, look up cards with `rarity: 'starter'` matching their character. Build deck as array of card IDs. Import card data from shared package.
b. **Starting HP**: Look up per character — ironclad: 75, silent: 70, defect: 75, watcher: 68
c. **Starter relics**: Each character gets their starter relic — ironclad: burning_blood, silent: ring_of_the_snake, defect: cracked_core, watcher: pure_water
d. **Solo relic**: If only 1 player, also add 'loaded_die' relic (check if it exists in relic data, if not create a constant)
e. **Generate map**: Call `generateMap(rng)`, store in a game-level state
f. **Select boss**: Pick random boss from `bossEnemies` array using rng
g. **Neow's Blessings**: Draw 1 event card per player from the events pool (shuffle and deal). Store as pending resolution (players will choose options). For now, store drawn Neow cards in game state as `neowBlessings: string[]` per player. Full resolution is a Phase 5 concern — for now just draw and store.
h. **Build initial CombatGameState**: Create state with all players, their decks, HP, relics, empty hands. Set phase to 'PLAYER_ACTIONS', round to 0.
i. **Set up first encounter**: The first node on the map is an encounter. Pick enemy card(s) for the first encounter from `encounterEnemies` (1 per player). Call `initCombat()` to set up the first fight.
j. **Store**: Set `room.gameState = initializedState`

Return the initialized CombatGameState. The caller (server message handler) will broadcast STATE_UPDATE.

Note: The map and boss data will be stored alongside CombatGameState. Since CombatGameState doesn't have map fields, create an extended type or store map/boss on the Room object directly:
```typescript
room.map = generateMap(rng);
room.bossId = randomBoss.id;
room.neowBlessings = drawnNeowCards;
```
