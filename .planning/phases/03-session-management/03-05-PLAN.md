1. **actionQueue.ts**: Create a per-room FIFO action serializer.

Export class `ActionQueue`:
- `private processing: boolean = false`
- `private queue: Array<() => void> = []`
- `enqueue(fn: () => void): void` — adds to queue, calls `processNext()` if not already processing
- `private processNext(): void` — sets processing = true, dequeues and executes next fn, then calls processNext() again. Sets processing = false when queue empty.

This ensures actions for a room execute sequentially even if multiple WebSocket messages arrive simultaneously. Since processAction is synchronous, this is a simple serial queue — no async/await needed.

2. **gameHandlers.ts**: Export game action handler functions.

Import `processAction` from engine, `ActionQueue` from actionQueue.

Export `createGameHandlers()` that returns handlers:

- **handlePlayCard(room, playerId, cardId, targetIds?):**
  - Validate room.gameState exists (error if not)
  - Enqueue on room's action queue:
    - Build Action: `{ type: 'PLAY_CARD', playerId, cardId, targetId: targetIds?.[0] }`
    - Call `processAction(room.gameState, action)`
    - Update `room.gameState` with result
    - Return new state for broadcasting

- **handleEndTurn(room, playerId):**
  - Validate room.gameState exists
  - Enqueue: Build `{ type: 'END_TURN', playerId }`, process, update state
  - If player had a disconnect timer, clear it

- **handleChat(room, playerId, text):**
  - Just broadcast CHAT_MESSAGE to all connections (no game state change)

- **broadcastState(room):**
  - For each connected WebSocket in room.connections:
    - Create per-player view: redact other players' drawPile (replace with array of same length but empty strings)
    - Send STATE_UPDATE with the redacted game state

- **broadcastLobby(room):**
  - Send LOBBY_UPDATE to all connections with room.lobby

Each room gets its own ActionQueue instance (stored on Room object or managed by gameHandlers).
