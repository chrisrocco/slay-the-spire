  <name>Task 1: Implement reconnection handler</name>
  <files>packages/server/src/rooms/reconnection.ts</files>
  <action>
Export functions for reconnection management:

1. **handleReconnect(roomManager, roomCode, token, ws):**
   - Look up room by code. If not found: return `{ error: 'Room not found or expired' }`
   - Find playerId whose reconnection token matches. If no match: return `{ error: 'Invalid reconnection token' }`
   - Clear any disconnect timer for this player: `clearTimeout(room.disconnectTimers.get(playerId))`
   - Update room.connections: set playerId -> ws
   - Update room.lastActivity to Date.now()
   - Build reconnection payload:
     - If game is in progress (room.gameState !== null): send STATE_UPDATE with full game state + last 20 combat log entries
     - If still in lobby: send LOBBY_UPDATE with current lobby state
   - Return `{ playerId, lobby: room.lobby, gameState: room.gameState }`

2. **handleDisconnect(room, playerId):**
   - Remove from room.connections
   - Update room.lastActivity
   - Do NOT remove from lobby.players (they keep their slot)
   - If game is in progress: start a 30-second disconnect timer. When timer fires, auto-signal END_TURN for this player (the actual END_TURN processing is handled by Plan 05's game handlers — for now store a callback or flag)
   - Store timer in room.disconnectTimers

3. **isPlayerConnected(room, playerId): boolean**
   - Check if playerId is in room.connections and the WebSocket is open

4. **getReconnectionPayload(room, playerId): ServerMessage**
   - If room.gameState exists: return STATE_UPDATE with game state, trimming combatLog to last 20 entries for the reconnecting player
   - If lobby only: return LOBBY_UPDATE with current lobby state

Note: Reconnection tokens are generated in Plan 01 when players first connect. This plan handles the reconnection flow using those existing tokens.
