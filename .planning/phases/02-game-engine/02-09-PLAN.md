  <files>packages/server/src/game/engine/combat.ts, packages/server/src/game/engine/index.ts</files>
  <action>
  1. **Define Action type** (in `index.ts` or `combat.ts`):
     ```typescript
     export type Action =
       | { type: 'START_COMBAT'; playerIds: string[]; enemyIds: string[]; playerCount: number }
       | { type: 'PLAY_CARD'; playerId: string; cardId: string; targetId?: string }
       | { type: 'USE_POTION'; playerId: string; potionId: string; targetId?: string }
       | { type: 'USE_MIRACLE'; playerId: string }
       | { type: 'USE_SHIVS'; playerId: string; targetId: string }
       | { type: 'END_TURN'; playerId: string }
       | { type: 'EVOKE_ORB'; playerId: string; orbIndex: number }
       | { type: 'RESOLVE_SCRY'; playerId: string; discardIds: string[] }
     ```

  2. **Create combatReducer** (`combat.ts`):
     - `combatReducer(state: CombatGameState, action: Action, cardLookup, enemyLookup): CombatGameState`
     - Switch on `action.type`:
       - `START_COMBAT`: Initialize combat state — set up enemy combat states from enemy data, assign enemies to rows, set phase to first player turn, call `startPlayerTurn`
       - `PLAY_CARD`: Validate phase is PLAYER_ACTIONS, call `playCard`
       - `END_TURN`: Call `signalEndTurn`. If all players ended, the signalEndTurn function handles transition.
       - `USE_MIRACLE`: Call `useMiracle` from watcher.ts
       - `USE_SHIVS`: Call `resolveShivs` from silent.ts
       - `EVOKE_ORB`: Call `evokeOrb` from defect.ts
       - `RESOLVE_SCRY`: Call `resolveScry` from deck.ts
       - `USE_POTION`: Stub for Phase 5 (return state unchanged with log)
     - After enemy turn resolves (phase transitions from ENEMY_TURN):
       - Check if combat is over (all enemies dead or player dead)
       - If not over: automatically call `startPlayerTurn` for next round
     - Phase transition logic:
       - PLAYER_ACTIONS: accept PLAY_CARD, END_TURN, USE_* actions
       - After all END_TURN: run end-of-turn triggers + poison + discard → set phase to ENEMY_TURN
       - ENEMY_TURN: automatically resolve all enemy actions → check deaths → if combat continues, start new player turn
       - COMBAT_END: reject all actions

  3. **Create initCombat** (`combat.ts`):
     - `initCombat(state: GameState, enemyCards: EnemyCard[], playerCount: number): CombatGameState`
     - Hydrate GameState into CombatGameState:
       - Add combat tokens to each player (all zeros/defaults)
       - Create EnemyCombatState for each enemy (HP resolved from EnemyHP by playerCount)
       - Assign enemies to rows (encounter: 1 per player row, elite: shared, boss: all rows)
       - Set round to 1, phase to PLAYER_ACTIONS
       - Call startPlayerTurn for the initial turn

  4. **Create processAction** (`index.ts`):
     - `processAction(state: CombatGameState, action: Action): CombatGameState`
     - This is the PUBLIC API. It wraps combatReducer with the card/enemy lookup functions built from the shared data.
     - Import card data and enemy data from `@slay-online/shared`
     - Build cardLookup from card data maps
     - Build enemyLookup from enemy data maps
     - Call combatReducer
     - Return result

  Use `.js` extensions in all imports.
