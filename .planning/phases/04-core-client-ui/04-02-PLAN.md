Create `packages/client/src/utils/cardLookup.ts`:

1. Import all card arrays from `@slay-online/shared`: `ironcladCards`, `silentCards`, `defectCards`, `watcherCards`.
2. Import the `PlayerCard` type.
3. Build a `Map<string, PlayerCard>` at module load time from all card arrays.
4. Export `getCard(id: string): PlayerCard | undefined` — lookup by card ID.
5. Export `getCardOrPlaceholder(id: string): PlayerCard` — returns the card or a placeholder object with `{ id, name: 'Unknown Card', character: 'colorless', rarity: 'common', type: 'Attack', cost: 0, text: '???', upgraded: false, keywords: [] }` for safety.
6. <task type="auto">
  <name>Task 2: Build Card and CardTooltip components with board game styling</name>
  <files>packages/client/src/components/combat/Card.tsx, packages/client/src/components/combat/Card.module.css, packages/client/src/components/combat/CardTooltip.tsx, packages/client/src/components/combat/CardTooltip.module.css</files>
  <action>
1. Create `Card.module.css`:
   - `.card` — board game card style: 120px wide, ~170px tall, rounded corners, dark card body (`#2a2a3e`), border that uses a CSS variable `--char-color` for character theming.
   - `.header` — title bar at top with character color background, card name in white.
   - `.cost` — energy cost circle in top-left corner, gold background.
   - `.artPlaceholder` — middle area, dark gradient placeholder (no images in this phase).
   - `.text` — effect text area below art, smaller font, padding.
   - `.type` — card type label at bottom (Attack/Skill/Power), small italic.
   - `.keywords` — keyword badges (Exhaust, Ethereal, etc.) shown below text if present.
   - `.selected` — bright glow border (gold/white), slight scale-up (`transform: scale(1.05)`), z-index bump.
   - `.dimmed` — opacity 0.5, grayscale filter, pointer-events: none.
   - `.card:hover` — slight lift (`translateY(-8px)`), transition 200ms.
   - Use CSS custom properties from variables.module.css via inheritance.

2. Create `Card.tsx`:
   - Props: `{ cardId: string; selected?: boolean; playable?: boolean; onSelect?: (cardId: string) => void; onHover?: (cardId: string | null) => void }`.
   - Use `getCardOrPlaceholder(props.cardId)` to resolve card data.
   - Character color mapping: `{ ironclad: '#c62828', silent: '#2e7d32', defect: '#1565c0', watcher: '#6a1b9a', colorless: '#666' }`.
   - Render: card div with `--char-color` CSS variable, header with name, cost circle, art placeholder, text, type label.
   - If card has keywords (exhaust, ethereal, retain, innate), show as small badges.
   - Apply `styles.selected` when `props.selected` is true.
   - Apply `styles.dimmed` when `props.playable` is false.
   - onClick calls `props.onSelect?.(props.cardId)`.
   - onMouseEnter calls `props.onHover?.(props.cardId)`, onMouseLeave calls `props.onHover?.(null)`.

3. Create `CardTooltip.module.css`:
   - `.tooltip` — enlarged card view: 240px wide, ~340px tall (2x the normal card). Position absolute, z-index 100, box-shadow for elevation. Same card layout structure but larger text, more readable.
   - `.visible` — display: block with fade-in animation (opacity transition 200ms).
   - Dark background, same board game card structure as Card but bigger.

4. Create `CardTooltip.tsx`:
   - Props: `{ cardId: string | null; anchorRect?: { x: number; y: number } }`.
   - Use `<Show when={props.cardId}>` to conditionally render.
   - When visible, use `getCardOrPlaceholder` to get card data.
   - Position near the anchor rect (above the card, centered). Use `style={{ left: ..., top: ... }}` with position fixed.
   - Render the same card layout as Card but at 2x size with full text clearly readable.
   - Ensure tooltip stays within viewport bounds (clamp position).
  </action>
