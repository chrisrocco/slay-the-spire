  <files>packages/client/src/utils/enemyLookup.ts, packages/client/src/components/combat/EnemyCard.tsx, packages/client/src/components/combat/EnemyCard.module.css</files>
  <action>
1. Create `packages/client/src/utils/enemyLookup.ts`:
   - Import `encounterEnemies`, `eliteEnemies`, `bossEnemies` from `@slay-online/shared`.
   - Import `EnemyCard` type from `@slay-online/shared`.
   - Build a `Map<string, EnemyCard>` from all enemy arrays at module load.
   - Export `getEnemy(id: string): EnemyCard | undefined`.
   - Export `getEnemyOrPlaceholder(id: string): EnemyCard` — returns placeholder with `{ id, name: 'Unknown Enemy', ... }`.

2. Create `EnemyCard.module.css`:
   - `.enemyCard` — dark card style, ~160px wide, ~200px tall. Rounded corners, border. Enemy portrait placeholder area.
   - `.header` — enemy name at top, white text on dark background.
   - `.hpBar` — HP bar: red fill on dark background, shows current/max HP text overlaid.
   - `.block` — blue-gray shield icon with block value, shown only when block > 0.
   - `.tokens` — small colored circles for status tokens (vulnerable=orange, weak=green, strength=red, poison=purple).
   - `.dead` — grayed out, flipped effect (opacity 0.3, rotateY or cross-out).
   - `.targetable` — glowing border, cursor pointer, indicating this enemy can be targeted.
   - `.boss` — larger size, special border treatment for boss enemies.
   - CSS transitions (200ms) on HP bar width changes.

3. Create `EnemyCard.tsx`:
   - Props: `{ enemyId: string; combatState?: { hp: number; maxHp: number; block: number; isDead: boolean; vulnerableTokens: number; weakTokens: number; strengthTokens: number; poisonTokens: number }; targetable?: boolean; onClick?: (enemyId: string) => void }`.
   - Use `getEnemyOrPlaceholder(props.enemyId)` for static data (name, category).
   - Render: name header, HP bar (percentage fill = combatState.hp / combatState.maxHp), block shield (if > 0), status token indicators.
   - If `props.combatState?.isDead` — apply dead style.
   - If `props.targetable` — apply targetable style, onClick calls `props.onClick`.
   - Boss enemies get `.boss` class (larger).
   - Status tokens shown as small colored dots with count: Vulnerable (orange), Weak (green), Strength (red, show as +N), Poison (purple).
   - Note: The server broadcasts the full CombatGameState which includes `enemyCombatStates` as a Record. The game state received by the client will have this field even though the base GameStateSchema doesn't declare it — the data is present in the JSON. The client should access `(state.game as any).enemyCombatStates?.[enemyId]` or define a local type extension.
  </action>

  <files>packages/client/src/components/combat/EnemyZone.tsx, packages/client/src/components/combat/EnemyZone.module.css, packages/client/src/components/combat/SharedInfo.tsx, packages/client/src/components/combat/SharedInfo.module.css, packages/client/src/components/combat/CombatEnd.tsx, packages/client/src/components/combat/CombatEnd.module.css</files>
  <action>
1. Create `EnemyZone.module.css`:
   - `.enemyZone` — top area of combat, flex row with wrap, centered, gap between enemies. Enemies arranged by row (top row first).
   - `.rowLabel` — small label "Row 1", "Row 2" etc. if multiple rows.

2. Create `EnemyZone.tsx`:
   - Props: `{ activeEnemies: string[]; enemyCombatStates: Record<string, any>; selectedCard: string | null; onEnemyClick: (enemyId: string) => void }`.
   - Use `<For each={props.activeEnemies}>` to render EnemyCard components.
   - For each enemy, pass combat state from `props.enemyCombatStates[enemyId]`.
   - An enemy is `targetable` when `props.selectedCard !== null` and the enemy is not dead. (Simplified targeting — all alive enemies are valid targets when a card is selected. Server validates actual legality.)
   - Filter out dead enemies from display OR show them grayed out (show grayed out to match "dead enemies flip over" board game rule).

3. Create `SharedInfo.module.css`:
   - `.sharedInfo` — center strip between enemies and player board. Dark semi-transparent background, horizontal flex, centered items.
   - `.dieResult` — large die face display. White text on dark background. Number inside a die-shaped box (rounded square). Size ~60px.
   - `.turnPhase` — phase name text: "Player Actions", "Waiting for Players", "Enemy Turn", "Combat Over".
   - `.round` — round counter: "Round N".
   - Brief CSS transition (200ms) on die result changes.

4. Create `SharedInfo.tsx`:
   - Props: `{ dieResult: number | null; phase: TurnPhase; round: number; players: Array<{ id: string; nickname: string; endedTurn: boolean }> }`.
   - Render: Die result (show "--" if null), turn phase label (human-readable), round counter.
   - Show player end-turn status: each player's nickname with a green checkmark ✓ when `endedTurn` is true, gray circle ○ when not. This satisfies the "all players can see who has and hasn't ended their turn" requirement (UI-06 support).
   - Phase label mapping: `PLAYER_ACTIONS` → "Player Actions", `WAITING_FOR_ALL_PLAYERS` → "Waiting for Players...", `ENEMY_TURN` → "Enemy Turn", `CLEANUP` → "Cleanup", `COMBAT_END` → "Combat Over".

5. Create `CombatEnd.module.css`:
   - `.overlay` — fixed position, full screen, dark semi-transparent background (rgba(0,0,0,0.8)), z-index 200, flex center.
   - `.modal` — centered card, dark background, border, rounded corners, padding. Max-width 400px.
   - `.result` — large text: "Victory!" (green) or "Defeat" (red).
   - `.returnButton` — prominent button, styled like EndTurnButton but larger. "Return to Lobby".
   - Fade-in animation on appearance.

6. Create `CombatEnd.tsx`:
   - Props: `{ gameState: GameState; onReturnToLobby: () => void }`.
   - Show only when `props.gameState.phase === 'COMBAT_END'`.
   - Determine win/loss: Check if all enemies are dead (win) or any player is dead (loss). Use `enemyCombatStates` if available; otherwise check `activeEnemies.length === 0` as a proxy for win.
   - Simpler approach: if phase is COMBAT_END and at least one player has HP > 0, it's a win. If all players have HP 0, it's a loss.
   - Display result text and "Return to Lobby" button.
   - "Return to Lobby" calls `props.onReturnToLobby` which could reload the page or reset the store to lobby phase.
